# Starting knowledge and ZPD floor (2026-07-03)

Established at kickoff, before any lesson was completed. Steers how lessons are pitched.

- **Ordinary dependent type theory: solid.** Knows the identity type and refl already. Do not re-teach MLTT basics; build HoTT as a reinterpretation of what they have.
- **Proof assistants: read, not written.** Can follow Agda/Coq/Lean snippets. Use code as illustration only, never as a required exercise.
- **Topology: "vague memory."** Has heard of continuous deformation, nothing rigorous. Spatial intuition (paths, homotopy, components) must be built explicitly, not assumed.
- **Category theory: knows the basics.** Groupoid / functor / "structure-preserving" framing is a fast on-ramp. Lesson 1 leaned on "a groupoid is a category with all arrows invertible" and it should land.
- **Univalence: cannot yet state it.** This is a named target in [[MISSION.md]]. Building toward it across lessons; do not front-load it.

Implication: the course is knowledge-heavy (conceptual) more than skills-heavy. Retrieval practice targets *explaining in own words*, not writing proofs.
