h-levels: How Tall Is the Tower?

Lesson 1 said the path tower can climb forever. Most types stop early. h-levels are the ruler that says how far up a type stays interesting before it goes trivial.

The win for this lesson Given a type, you can name which rung it sits on, say what that buys you, and explain why the universe of sets sits exactly one rung above the sets it contains. This is the vocabulary every HoTT proof leans on: "assume A is a set," "P is a proposition."

The ladder, from the bottom

Each rung is defined by how trivial its equality is. Read these as "how much structure is left."

-- Rung 0: contractible. One point, and everything equals it.
isContr A = Σ (c : A). (x : A) -> c = x

-- Rung 1: proposition. At most one point: any two are equal.
isProp  A = (x y : A) -> x = y

-- Rung 2: set. Equality is trivial: any two paths are equal.
isSet   A = (x y : A) -> isProp (x = y)

-- Rung 3: groupoid. Paths can differ, but 2-paths are trivial.
isGpd   A = (x y : A) -> isSet  (x = y)

Notice the pattern: each rung says "all my identity types sit one rung lower." That is the whole definition, stated once:

hLevel 0       A = isContr A
hLevel (suc n) A = (x y : A) -> hLevel n (x = y)

So proposition is hLevel 1, set is hLevel 2, groupoid is hLevel 3, and it continues. To go up one rung, look at the identity types and check they are one rung simpler.

A numbering warning for when you read the book. This lesson uses the Agda/1Lab convention: contractible = 0, prop = 1, set = 2. The HoTT Book shifts everything down by two, so contractible is a (-2)-type, prop is (-1), set is a 0-type. Same ladder, different zero. Do not let the offset trip you up.

Levels are cumulative: a type at rung n is also at every rung above it. A set is trivially a groupoid too. The rung of a type is the lowest one it satisfies.

Anchor 1: "set" is where the maths you know lives

A set is a type whose equality is a mere proposition: given two elements, there is at most one path between them, so equality is a plain yes/no fact again. That is exactly the assumption you were carrying in ordinary type theory without naming it. So the slogan from earlier lands precisely:

UIP, restated UIP (every equality proof is unique) is the assumption that every type is a set. It caps the whole ladder at rung 2. HoTT drops that cap, which is why the tower can be taller.

Anchor 2: the universe sits one rung up

Here is the loop back to Lesson 2. Bool is a set: its two points have no interesting paths, and true = false is empty. But the universe that contains Bool is not a set. Under univalence, Bool =𝒰 Bool has two distinct points (refl and ua not), so the universe's equality is not a proposition. The universe of sets is a groupoid, rung 3.

The pattern to keep Going "up a type" often means going "up a rung." The elements of Bool live at rung 2; Bool itself, as an element of the universe, lives one level higher. That climb is exactly the ∞-groupoid tower from Lesson 1, now with a ruler on it.

Why bother classifying? What each rung buys you

The rung tells you how much you have to do. The cleanest payoff is at rung 1: 

-- If you have shown A is a proposition, equalities come for free:
pf : isProp A
-- then for ANY x y : A,   pf x y : x = y     -- no path to construct

-- If A is only a set (or worse), you must build the path yourself,
-- because there can be genuinely different elements to tell apart.

So "A is a proposition" means you never worry about which proof of A you hold: any two are equal, so proofs are irrelevant. "A is a set" means you can reason with plain equality and forget higher paths. The h-level is a budget: it tells you the most path structure a type can carry, so you only ever carry that much. That is why HoTT proofs open with "assume A is a set" or "since P is a prop" and everything downstream gets simpler.

A concrete case where you invoke it

pf : isProp A is a function: unfolded it is (x y : A) → x = y, so you feed it two elements of A and it hands back a path between them. The everyday place you reach for it is when you are proving two bundles equal and one field is a proof. Take "a natural number that is at most ten": 

Bounded = Σ (n : ℕ). (n ≤ 10)   -- a number, plus a proof it is ≤ 10

-- Two bundles, same number, two different-looking proofs:
--   (7, le1)   and   (7, le2)     where  le1 le2 : 7 ≤ 10
-- To show (7, le1) = (7, le2) you owe: 7 = 7 (refl) AND le1 = le2.
-- The second is free, because (7 ≤ 10) is a proposition:
pf : isProp (7 ≤ 10)
pf 7-le1 7-le2 : le1 = le2       -- no proof to build; pf produces it

So knowing ≤ is a proposition is exactly what lets you treat a bounded number like just its number. The slogan: props are what make Σ (a : A). P a behave like a subset of A rather than a type carrying extra data, because the proof component is automatically equal.

Your "swap in a logic world" instinct has a name. Getting A = B between two propositions from the fact that they imply each other (A ↔ B) is propositional extensionality, the rung-1 shadow of univalence. Since A and B are props, which proof you have never matters, so you swap one for the other freely. Example: (n is even) ↔ (n mod 2 = 0), so the two are equal and anything proved about one transfers. That is Lesson 2's whole story, flattened to the case where proofs are irrelevant.

Check yourself

Answer from memory first. The click gives immediate feedback.

What does it mean for a type to be a set (rung 2)?

A set has trivial equality: for any two points, all paths between them are equal, so its identity types are propositions. Points can still be distinct; it is the paths that collapse.

Why is the universe of sets a groupoid rather than a set?

Univalence gives Bool = Bool two distinct inhabitants (refl and ua not), so the universe's equality is not a proposition. Its identity types are sets, which puts the universe itself one rung higher.

Say it in your own words

A colleague asks: "What is an h-level, and why does it matter that Bool is a set but the universe is a groupoid?" Answer in three sentences.

An h-level measures how far up a type's path tower stays non-trivial before every higher path collapses: contractible, then proposition, then set, then groupoid, and up. Bool is a set because its points are distinct but their equality is trivial, whereas the universe is a groupoid because univalence gives type-equalities like Bool = Bool genuinely different proofs, so its equality is not trivial. It matters because the rung is a budget: knowing a type is a proposition or a set tells you how little path structure you must track, which is exactly what makes proofs manageable.
