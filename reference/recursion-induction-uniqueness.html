Recursion, Induction, and Uniqueness

Why uniqueness is the prize, what breaks without it, the levels it comes in, and how "there exists" gets turned into "for all." A companion to Induction as Initiality.

The through-line Contractible = unique existence = ∃! = the meaning of the word "the." Uniqueness buys two things: you can name "the X" (well-defined), and you can define an operation by one canonical case (no choices). When it fails, you are forced to choose — and a choice is exactly what makes things ambiguous, non-canonical, or ill-defined.

1. What uniqueness buys, across mathematics

The same phenomenon everywhere: unique existence is what makes "the thing" and lets you define by cases.

SettingContractible (∃!) → goodNot contractible → breaks
Function vs relationeach input has one output → a function you can applymultivalued → not a function; you must pick an output
"The X" (definite description)∃! x. P x → "the P" is meaningfulnone or many → "the P" is ill-formed ("the king of France")
Recursion on syntaxunique parse → define eval / subst by recursionambiguous grammar → the recursive definition isn't well-defined (3-2-1)
Matrix inverseevery b has one solution to Ax=b → A⁻¹ existssingular → none or many solutions → no inverse
Universal propertyunique mediating map → object pinned up to unique isonon-unique → fails to characterize the object

Two of these are the same mechanism as HoTT's equivalences: a relation is a function when each input's fibre is a singleton, and a bijection when fibres are singletons both ways — that's the contractible-fibres definition of equivalence, and the matrix case (A invertible ⟺ unique preimage for every b) is identical.

2. The levels of uniqueness

"Unique" means different things at different places. Keep them apart.

LevelWhat it saysWhere it comes from
Single-valued (well-defined)one output per inputevery function has it; from unique parse / constructors
η / uniqueness of the maponly one function obeys the given equationsNOT from bare recursion; needs induction
Weakly initial vs initial∃ a map out (recursion) vs ∃! a map out (induction)the "!" is the η above, categorically
Prop-valued property (e.g. isEquiv)at most one witness that a thing has a propertycontractible fibres; a mere proposition
A function can be perfectly single-valued (Level 1, well-defined) while recursion still can't prove it is the unique solution to its equations (Level 2, η). Example: f 0 = 0, f(suc n) = f n + 2 defines 2n (single-valued, fine), but proving "no other function fits those equations" needs induction. Well-definedness ≠ η.
"Unique" means per-equations, not one-function-total 2n (f 0=0, f(suc n)=f n+2) and 3n (g 0=0, g(suc n)=g n+3) are different functions ℕ→ℕ, but they obey different equations. That is the existence axis (many recursion data → many functions), not a uniqueness failure: each is the unique solution of its own equations, so η holds for both. Uniqueness is only at stake when two functions obey the same equations.

Categorically: the recursion data (base, step) is a target algebra — an object. A₂ = (ℕ, 0, +2) and A₃ = (ℕ, 0, +3) are two different objects (same carrier ℕ, different structure map — an algebra is the whole triple). The functions 2n, 3n are the unique morphisms out of the initial algebra ℕ to A₂, A₃. So: different equations = different target objects = different unique morphisms. Uniqueness = one morphism per object (initiality); existence = many objects → many morphisms out of ℕ.

3. What breaks without uniqueness (the equivalence case)

If "is an equivalence" carried data instead of being a proposition, the junk is real. On the circle, certify id : S¹ → S¹ is an iso:

isIso(id) = Σ(g). (g = id) × (g = id)   -- both conditions say g = id
          ≃ (id = id) ≃ Π(x:S¹). x = x ≃ ℤ   -- a homotopy id=id winds n times

So isIso(id) ≃ ℤ: infinitely many distinct proofs that id is invertible. Four concrete costs:

  • You can't say "the inverse" — it depends which witness you picked.
  • Coherence tracking — every use carries and must respect the witness (setoid hell, one floor up).
  • An equivalence wouldn't be its function — same f, different equivalences.
  • Univalence miscounts — A ≃ A gains a spurious ℤ, can't match A = A, so (A=B) ≃ (A≃B) fails.

The "attack surface" is the identity type. With a prop: (f,w₁) = (g,w₂) ⟺ f = g (witnesses auto-agree, drop out). With data: also need w₁ = w₂ — more to prove, more ways to differ. Making it a prop is the same subtype mechanic as Σ(n). n ≤ 10 = "just a number."

4. Induction, and turning "exists" into "for all"

Goal of induction on P : ℕ → Type: from base : P 0 and step : (n) → P n → P(suc n), build a dependent function (n) → P n. That dependent function is a section of a bundle:

fibres P n over ℕ:            two arrows between E and ℕ:
   P0  P1  P2  P3                    E
    ·   ·   ·   ·                 s ↑ │ π
 ───0───1───2───3── ℕ              │ ▼
E = Σ(n). P n  (total space)         ℕ

π : E → ℕ,  π(n,x) = n      DOWN. plain function (first projection).
s : ℕ → E,  π∘s = id        UP. a SECTION — lands in the right fibre.
                            a section IS a dependent function (n) → P n = the "for all".
The argument: recursion + uniqueness ⟹ induction 
make E an algebra:   e_zero = (0, base),   e_suc (n,x) = (suc n, step n x)
recursion:           f : ℕ → E    (unique hom out of the initial algebra ℕ)
π is a hom, so π∘f : ℕ→ℕ obeys the same equations as id_ℕ
UNIQUENESS (ℕ initial ⇒ one hom ℕ→ℕ):   π∘f = id
   ⟹ π(f n) = n ⟹ f n = (n, d n) ⟹ f is a SECTION ⟹ d : (n) → P n  = induction

Why the detour through E: recursion only builds non-dependent maps ℕ → X. Packaging the moving target P into one type E = Σ(n).P n turns the dependent goal into a non-dependent one recursion can hit; uniqueness then forces the map to be a section (no landing in a wrong fibre — "no junk"). In initial-algebra language: f is the unique hom from the initial algebra into E, and initiality's uniqueness (one hom ℕ→ℕ) makes π∘f = id. See Induction as Initiality.

5. How the even numbers fail (the safety mechanism)

Try induction with P n = "n is even":

base : "0 is even"                    ✓
step : (n) → "n even" → "n+1 even"     ✗  IMPOSSIBLE (even + 1 is odd)

You cannot build step, so induction never starts — there is no f, so "f 1 has nothing to map to" never even arises. This is the theory refusing to prove the false statement "every number is even." The three views coincide:

E = Σ(n)."n even" ≅ evens   has a GAP over every odd n
a section needs a point over EVERY n (incl. n=1)   → impossible
no section ⟺ no (n)→"n even" ⟺ no step ⟺ the statement is false

6. Recursion is degenerate induction

Recursion is induction with a constant motive (the target doesn't vary with n):

induction:  (P : ℕ→Type) → P 0 → ((n)→P n→P(suc n)) → (n)→P n
recursion:  set P n := X:      X   → ((n)→X→X)        → ℕ→X

So a plain function ℕ → X is induction where P n is constant. And "recursion without uniqueness" is not another world — it is the weaker property, weakly initial (∃ a map out) versus initial (∃! a map out). We define ℕ as the initial algebra precisely so uniqueness comes for free, which is exactly what upgrades recursion to induction.

propertygives
weakly initial∃ hom outrecursion (no guaranteed uniqueness)
initial∃! hom outrecursion + uniqueness = induction

These come apart: weakly-initial-but-not-initial algebras exist, and there are models with the recursor but not induction. So "recursion alone" is a real, inhabited, strictly weaker notion — which is why the "!" is in the definition of an inductive type on purpose.

To be expanded on request. Everything here dualizes. Inductive types = initial algebras (least fixed points, built by constructors, consumed by recursion/induction). Coinductive types = terminal coalgebras (greatest fixed points, produced by "unfolding"/corecursion, observed by destructors). Example: streams (infinite lists) are the terminal coalgebra of F(X) = A × X — you never finish building one, you only observe head/tail. The uniqueness dualizes too: terminal = ∃! map into it, and the coinductive "equality" is bisimulation. Ask and we will spell out the induction↔coinduction mirror in full.
