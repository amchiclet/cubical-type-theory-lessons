Cubical Type Theory Lessons · Glossary

Glossary

One line per term. The canonical vocabulary for this course — every lesson uses these words in exactly this sense. "Lesson" marks where the term is first taught.

Reading guide: the left column is the type theory word; the "means" column gives the spatial reading. Learning to switch fluently between the two columns is most of what "understanding HoTT" is.

The core dictionary

TermMeansLesson
TypeBehaves as a space up to homotopy, equivalently an ∞-groupoid (see below). Its inhabitants are points; it also carries all the higher path structure. "Space" is the intuition; "∞-groupoid" is what is precisely there.0001
Space (as used here)A topology word, used in HoTT by interpretation, not definition. A type carries no point-set "nearness," only paths. Precisely it is a homotopy type = a space viewed only up to continuous deformation.0001
Homotopy type / homotopy hypothesisA homotopy type is a space up to deformation. Grothendieck's homotopy hypothesis: homotopy types are ∞-groupoids. So "type = space" and "type = ∞-groupoid" say the same thing.0001
Synthetic vs analyticAnalytic (classical topology): build spaces from points + nearness, derive paths, then forget nearness. Synthetic (HoTT): paths are primitive; there is no underlying nearness to forget. Cubical is a synthetic, computational take.0001
Term / element a : AA point of the space A.0001
Identity type a =A bThe space of paths from point a to point b. Its own points are proofs that a equals b.0001
Identification / path p : a =A bOne specific way a and b are equal; a path from a to b.0001
reflThe constant path that stays at a point; the canonical proof a =A a.0001
Higher pathA path between paths (p =a=b q), and paths between those, forever.0001
HomotopyA continuous deformation of one path (or function) into another; a higher path.0001
GroupoidPoints + invertible arrows between them, with composition, inverses, identities. A category where every arrow is iso.0001
∞-groupoidPoints, paths, paths-between-paths, … all the way up. The full structure every type has.0001

TermMeansLesson
h-level / truncationHow tall a type's path-tower is: contractible < proposition < set < groupoid < …0002 (planned)
UIP (uniqueness of identity proofs) / axiom KAny two proofs of a = b are themselves equal. Equivalently: every type is a set. HoTT drops it — it contradicts univalence.0002 (planned)
Judgmental equality a ≡ bSameness the typechecker decides by computation. Distinct from propositional equality a = b (a type of proofs).0003 (planned)
Intensional vs extensional TTIntensional: ≡ and = stay separate, typechecking decidable. Extensional: equality reflection collapses them (implies UIP, undecidable). HoTT is intensional.0003 (planned)
Function extensionality (funext)If f x = g x for all x, then f = g. Not provable in bare HoTT; an axiom or a consequence of univalence; a computing theorem in cubical.0007 (planned)
Proposition (mere)A type with at most one point up to a path: any two proofs are equal. "Truth value."0002 (planned)
Set (0-type)A type whose paths are trivial: equality is a mere proposition. Ordinary maths lives here.0002 (planned)
apApply a function to a path: f sends a path a = b to a path f a = f b. Functoriality.0003 (planned)
transportMove a point along a path in the base to the fibre over the other end. Substitution, made spatial.0003 (planned)
Path induction (J)To prove something about all paths, it suffices to prove it for refl.0003 (planned)
Equivalence A ≃ BA function with a well-behaved inverse; the right notion of "same" for types.0004 (planned)

Representation independenceNo property can distinguish equivalent types, so a proof about one representation transfers to any equivalent one. False without univalence; a theorem with it.0002
Interval ICubical's primitive "line" with ends i0, i1. A path is a function I → A.0007 (planned)
