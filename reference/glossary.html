Glossary

One line per term. The canonical vocabulary for this course — every lesson uses these words in exactly this sense. "Lesson" marks where the term is first taught.

Reading guide: the left column is the type theory word; the "means" column gives the spatial reading. Learning to switch fluently between the two columns is most of what "understanding HoTT" is.

The core dictionary

TermMeansLesson
TypeBehaves as a space up to homotopy, equivalently an ∞-groupoid (see below). Its inhabitants are points; it also carries all the higher path structure. "Space" is the intuition; "∞-groupoid" is what is precisely there.0001
Space (as used here)A topology word, used in HoTT by interpretation, not definition. A type carries no point-set "nearness," only paths. Precisely it is a homotopy type = a space viewed only up to continuous deformation.0001
Homotopy type / homotopy hypothesisA homotopy type is a space up to deformation. Grothendieck's homotopy hypothesis: homotopy types are ∞-groupoids. So "type = space" and "type = ∞-groupoid" say the same thing.0001
Synthetic vs analyticAnalytic (classical topology): build spaces from points + nearness, derive paths, then forget nearness. Synthetic (HoTT): paths are primitive; there is no underlying nearness to forget. Cubical is a synthetic, computational take.0001
Term / element a : AA point of the space A.0001
Identity type a =A bThe space of paths from point a to point b. Its own points are proofs that a equals b.0001
Identification / path p : a =A bOne specific way a and b are equal; a path from a to b.0001
reflThe constant path that stays at a point; the canonical proof a =A a.0001
Higher pathA path between paths (p =a=b q), and paths between those, forever.0001
HomotopyA continuous deformation of one path (or function) into another; a higher path.0001
GroupoidPoints + invertible arrows between them, with composition, inverses, identities. A category where every arrow is iso.0001
∞-groupoidPoints, paths, paths-between-paths, … all the way up. The full structure every type has.0001

Dependent types, visually

A type family stands a type (a fibre) over each point of a base. Σ glues the fibres into one space; Π is a choice of one point in every fibre.

                fibres  P a  over the base A
    P a₀   P a₁   P a₂   P a₃
     ·      ·      ·      ·        the whole collection = TOTAL SPACE  Σ(a).P a
     ·      ·      ·      ·
   ──●──────●──────●──────●──  A
    a₀     a₁     a₂     a₃

element of Σ = ONE dot (a,x)        element of Π = a SECTION (a function)
     ·      ·      ·      ·              ·      ●      ·      ·
     ·      ●      ·      ·              ●      ·      ·      ●
     ·      ·      ·      ·              ·      ·      ●      ·
   ──●──────●──────●──────●──          ──●──────●──────●──────●──
     pick a base point + a               one dot in EVERY fibre;
     point in its fibre                  f : (a) ↦ f a ∈ P a
TermMeansLesson
Universe TypeA type whose elements are themselves types. Bool : Type. One level up.0005
Type family P : A → TypeA function returning a type for each element of A: a fibre over each point. A dependent type.0005
FibreThe type P a sitting over the point a. A whole type, not a single element.0005
Σ / total space Σ(a:A). P aAll fibres glued into one space. An element is a pair (a, x): a base point plus one point in its fibre. The "exists": witness + evidence.0005
Π / section Π(a:A). P aA dependent function f with f a : P a: one point in every fibre. The "for all". Written (a:A) → P a.0005
Recursion vs inductionRecursion builds a map into a fixed type (non-dependent); induction builds a section (dependent). Induction = recursion + uniqueness. See Recursion, Induction & Uniqueness.ref
Weakly initial vs initialWeakly initial: ∃ a map out (recursion). Initial: ∃! a map out (recursion + uniqueness = induction). "Recursion without uniqueness" is the weakly-initial corner.ref
Single-valued vs ηSingle-valued: one output per input (every function; well-defined). η: only one function obeys the equations (needs induction). Different "uniques".ref

Coming in later lessons

TermMeansLesson
h-level / truncationHow tall a type's path-tower is: contractible < proposition < set < groupoid < …0003
UIP (uniqueness of identity proofs) / axiom KAny two proofs of a = b are themselves equal. Equivalently: every type is a set. HoTT drops it, since it contradicts univalence.0003
Judgmental equality a ≡ bSameness the typechecker decides by computation. Not a type; there is no proof to hold. Distinct from propositional equality a = b (a type of proofs). Also called definitional equality.0004
Intensional vs extensional TTIntensional: ≡ and = stay separate, typechecking decidable. Extensional: equality reflection collapses them (implies UIP, undecidable). HoTT is intensional.0004
Equality reflectionThe rule that makes a theory extensional: from any proof p : a = b, conclude the judgment a ≡ b. Costs decidable typechecking and forces UIP, so HoTT declines it.0004
Function extensionality (funext)If f x = g x for all x, then f = g. Not provable in bare HoTT; an axiom or a consequence of univalence; a computing theorem in cubical.0009 (planned)
Proposition (mere)A type with at most one point up to a path: any two proofs are equal. "Truth value."0003
Set (0-type)A type whose paths are trivial: equality is a mere proposition. Ordinary maths lives here.0003
ContractibleA type with exactly one point up to a path: a centre c and a path c = x for every x. The bottom rung.0003
apApply a function to a path: f sends a path a = b to a path f a = f b. Functoriality.0005
transportMove a point along a path in the base to the fibre over the other end. Substitution, made spatial. Proves disequalities like true ≠ false by carrying a distinguishing property along a hypothetical path.0005
Path induction (J)To prove a property of every path, it suffices to prove it for refl. Justified because the based path space Σ(y). a = y is contractible with centre (a, refl); refl is the centre, not the only path.0005
Encode-decode / identity characterizationCompute a type's identity type by defining a Code that mirrors its structure and proving (x = y) ≃ Code x y with J. The identity type of a compound type is a formula in the identity types of its parts.0005
Equivalence A ≃ BA function with a well-behaved inverse; the right notion of "same" for types.0006 (planned)
UnivalenceIdentity of types is equivalence of types: (A =𝒰 B) ≃ (A ≃ B).0002, 0007 (planned)
Representation independenceNo property can distinguish equivalent types, so a proof about one representation transfers to any equivalent one. False without univalence; a theorem with it.0002
Interval ICubical's primitive "line" with ends i0, i1. A path is a function I → A.0010 (planned)
hcomp / transport (Kan ops)Cubical operations that let you compose and move paths; make everything compute.0011 (planned)
GlueThe cubical construction that turns an equivalence into a path, making univalence a theorem.0011 (planned)
